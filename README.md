# Колесо бонусов «Рабочие Руки» — гипотеза H-025

Игровая механика для приложения РР: исполнитель получает вращение колеса за каждое выполненное задание, а призы действуют, пока он не отказывается от принятых заданий. Запретов нет — отказ просто отменяет активные призы: значок репутации снимается, надбавки сгорают. Пряник вместо кнута.

## Смотреть онлайн

Презентация проекта: https://iriskazaraeva.github.io/rr-bonus-wheel/

Кликабельный прототип: https://iriskazaraeva.github.io/rr-bonus-wheel/prototype.html

## Как работает механика

Шаг 1. Выполнил задание — получил вращение. Копятся до 5, сгорают через 30 дней.

Шаг 2. Крутишь колесо — забираешь приз: баллы рейтинга со значком репутации, двойные баллы, деньги к следующей выплате, «10 заданий без комиссии», надбавка +150 ₽ на 5 заданий, мерч РР, джекпот 500 ₽.

Шаг 3. Отказался от принятого задания — активные призы отменяются. Отказ по уважительной причине решается через поддержку, призы сохраняются.

## Экономика

Вращение стоит компании ~93 ₽ — это 11,6% маржи с задания. 73% призов бесплатные: рейтинг, значок репутации, «ещё одно вращение». Денежные призы условные — платятся по мере выполнения заданий и сгорают при отказе. Мерч отправляется со складов в Москве и Краснодаре (~700 ₽ с доставкой), поэтому его вес всего 2%. Бюджет ограничен лимитом: при превышении колесо отключается фиче-флагом.

## Что в файлах

index.html — презентация для руководства: блок «что изменилось в версии 2», проблема в деньгах, механика, экономика с расчётом на 100 вращений, полная карточка гипотезы H-025, план проверки.

prototype.html — кликабельный прототип: три экрана приложения в фирменном стиле РР, включая симуляцию отказа с отменой призов, плюс кабинет маркетолога в CRM — настройка секторов с живым калькулятором бюджета, аналитика по периодам и карта эксперимента.

Обе страницы — чистый HTML/JS без зависимостей: открываются и по ссылкам выше, и локально двойным кликом.

## Гипотеза H-025 кратко

Если запустить колесо бонусов, призы которого действуют только при отсутствии отказов, то доля исполнителей без отказов вырастет на +10% относительно baseline, потому что исполнители получат регулярный игровой стимул принимать задания без отказов.
