# Колесо бонусов «Рабочие Руки» — гипотеза H-025

Игровая механика для приложения РР: исполнитель получает вращение колеса за каждое выполненное задание, а право крутить есть только у тех, кто не отказывается от принятых заданий. Пряник вместо кнута: за отказ не наказываем сильнее — исполнитель теряет накопленную привилегию.

## Смотреть онлайн

Презентация проекта: https://iriskazaraeva.github.io/rr-bonus-wheel/

Кликабельный прототип: https://iriskazaraeva.github.io/rr-bonus-wheel/prototype.html

## Как работает механика

Шаг 1. Выполнил задание — получил вращение. Копятся до 5, сгорают через 30 дней.

Шаг 2. Крутишь колесо — забираешь бонус: баллы рейтинга, значок «Топ-исполнитель», сторис недели, деньги к следующей выплате.

Шаг 3. Отказался от принятого задания — колесо замерзает на 7 дней, накопленные вращения сгорают. Форс-мажор решается через поддержку без заморозки.

## Экономика

Вращение стоит компании ~48 ₽ — это 6% маржи с задания. 85% призов бесплатные: рейтинг, значки, сторис уже есть в продукте. Денежные бонусы фиксированные и начисляются только со следующим выполненным заданием — вместе с новой маржой. Чтобы колесо окупилось, достаточно снизить отказы на 2 процентных пункта.

## Что в файлах

index.html — презентация для руководства: проблема в деньгах, механика, экономика, полная карточка гипотезы H-025, план проверки на 6 недель.

prototype.html — кликабельный прототип: три экрана приложения в фирменном стиле РР, включая симуляцию отказа и заморозки, плюс кабинет маркетолога в CRM — настройка секторов с живым калькулятором бюджета, аналитика по периодам и карта эксперимента.

Обе страницы — чистый HTML/JS без зависимостей: открываются и по ссылкам выше, и локально двойным кликом.

## Гипотеза H-025 кратко

Если запустить колесо бонусов, где участие доступно только при отсутствии отказов, то доля исполнителей без отказов вырастет на +10% относительно baseline, потому что исполнители получат регулярный игровой стимул принимать задания без отказов.
